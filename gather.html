    • Gather

    Design Question

    How can we organize potluck contributions to ensure unique complementary dishes?

    Concept

    Planning a potluck can be a stressful process. As a host it is difficult to create a balanced menu when guests are unaware of what others are bringing. Gather is a dynamic web application designed to allow guests sign up for and delete dishes from a menu in order to make potluck coordination easy.

    Implementation

    • Node backend
    • Mongo data store via mLab
    • Express web server

    Product

    In order to make Gather feel like a lifestyle choice a background picture showing a stylish event was selected. Users are invited to join rather then sign up or register to give a feeling of exclusivity.

    As the user registers they enter their name, dish, and catagorize it as an "appetizer", "main", or "dessert". The information is stored to a mongo data base and identified by it's unique ID. After the form is submitted, a card is posted above which allows them to delete their entry should they change their mind.

    The menu page creates a clear layout for guests and hosts to ensure that the dishes are complementary and properly distributed between catagories. The menu will loop through all the entry IDs and filter and display them in the correct catagory of "appetizer", "main", or "dessert.

    Although Gather is designed as a web application, the use case for mobile is extremely likely. CSS media queries were used to style the page for smaller devices. A hamburger menu replaced the navigation bar for usability.

    Responsive styling allowed for the signup cards to stack. The signup form was kept on the same page for consistency and ease of use.
