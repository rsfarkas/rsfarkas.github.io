    Design Question

    How might we ease the stress of the average NYC residents’ commute?

    Concept

    Commuting is stressful. Paying for the subway shouldn’t add to that stress. Long lines at subway kiosks, the panic that sets in while rummaging for a paper-thin card, breakage on lost or expired cards; the process of paying for a subway fare is broken.

    This proposal contemplates building a mobile app allowing commuters to ride the subway with just their mobile phones. This app will decrease congestion, make using public transit easier and remove some of the stress from the average commute.

    Implementation

    • -Mind map of the problem space
    • -Developed 3 profiles of people likely to use yMetro.
    • -Wireframe of Application
    • -Paper prototypes
    • -Recorded user-testing sessions and iteratively built prototypes on Marvel.

    The end product will involve two parts. First the MTA will deploy 3D barcode readers at all subway entrance points to replace their existing physical card readers. Second is the development of a mobile app that allows users to pay via 3D barcode.

    This project is focused on the mobile app which, though designed for iOS, below, could easily be ported to Android. The app would also need to be accompanied by a database and webserver to securely store and transmit data to and from the app.

    Product

    yMetro allows users to use an app to pay for a subway fare and add value to their MTA account.

    After authenticating, the user lands on the home screen above. On this screen the user has the ability to select from any of the Metro Cards that have an active balance on their account. For example, if the user selects their Unlimited Monthly Card they will see the following screen:

    This screen loads a 3D barcode unique to the user and the Metro Card on their account that they have selected.When presented in front of an MTA 3D barcode reader, the relevant user account is debited for the transaction amount. To add credits to an account, a user need only tap the money icon in the upper right hand corner of the home screen.

    A user can have up to two payment methods associated with their account, however, only the default account is used for payment. Users can easily toggle their default accounts on this screen.

    The Add Value screen lets the user load additional funds onto the card of their choice using their default payment method. Settings such as login credentials and the user’s default payment method can be modified on the User Settings screen which is accessed via the icon on the upper left of the home screen.
