    Design Question

    How might we create an interactive dating app that creates a new role set for communication, preventing users from abandoning their match criteria when filtering thorugh potentail matches?

    Concept

    In order to alter the way communication is usually made on dating applications, MotherBot will function as an intermediary, screening potential matches for suitability.

    MotherBot aims to prevent dates from selecting inappropriate matches by requesting preferences, and also weighing in.

    Implementation

    • -Node backend
    • -Express as middleware

    App Flow

    • -User1 fills in their preferences
    • -User2 arrives at the MotherBot chat page
    • -Motherbot chats with User2, and if she determines she likes them as a match, she provides User1's contact information

    Functionalty

    • -User input is stored in variables which are passed to the chatbot
    • -Using Regex, the bot is able to check if the chat text matches any of the key words
    • -If no key words are matched, the bot will send a 'not interested' response
    • -If the bot doesnt like the answer based off the Regex, it will reject the matches. For example, all matches must want children
